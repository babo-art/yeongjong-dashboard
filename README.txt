영종 상황판 실행 및 배포 안내 (v2)

1. 배포 파일
- index.html
- assets\
- vendor\
- templates\
- README.txt

2. 실행 방법
1) 압축을 풉니다.
2) index.html을 엽니다.
3) 최초 실행 시 카카오 JavaScript 키를 입력합니다.
4) templates\영종_상황판_표준시트_템플릿.xlsx 또는 같은 구조의 실제 데이터 파일을 준비합니다.
   표준 시트는 위험물업체 / 소방관서 / 기준지점 3개입니다.
5) 하단 도크의 [설정] 버튼을 눌러 [데이터 업로드]로 엑셀 파일을 선택합니다.

3. 화면 구조
- 지도는 전체 화면을 사용합니다.
- 하단 도크에는 [설정] 버튼 1개만 있습니다.
- 지도에서 개체를 선택하면 하단 중앙의 selection HUD에 요약 정보가 표시됩니다.
- HUD는 정의 목록(라벨/값 2열) 표 형태이며 닫기 ✕ 버튼은 우상단입니다.
- 분석 메뉴는 폐기되었습니다.

4. 마커 클릭 동작
4-1. 빈 상태에서 한 번 클릭(anchor 지정)
- 기준지점(산) 또는 위험물제조소등을 클릭하면 그 마커가 anchor가 됩니다.
- 기준지점 클릭 시: 최근접 위험물제조소등 + 최근접 소방관서까지의 동심원/연결선/거리 라벨을 자동 표시합니다.
- 위험물제조소등 클릭 시: 최근접 소방관서까지의 동심원/연결선/거리 라벨을 자동 표시합니다.
- 소방관서를 클릭하면 단순 정보만 표시되며 anchor는 잡히지 않습니다(소방관서는 anchor 불가).
  selection HUD에는 [관할 위험물제조소등 N개]도 표시됩니다(공간 매칭 기반).

4-2. anchor 활성 상태에서 다른 마커 클릭(페어 선택)
- 두 마커 사이에 직선 1개와 거리 라벨 1개가 표시됩니다.
- 자동 nearest 그래픽은 사라지고 페어 라인 1개로 대체됩니다.
- HUD는 [anchor 이름 → target 이름] 제목과 직선거리 표로 변경됩니다.

4-3. 토글/해제
- anchor 또는 target으로 선택된 마커를 다시 클릭하면 전체 해제됩니다.
- 페어 상태에서 선택되지 않은 다른 마커를 클릭하면 새 target으로 교체됩니다(anchor는 유지).
- HUD 우상단 ✕ 버튼으로도 전체 해제 가능합니다.

5. 라벨 표시 규칙
- 산: 줌 레벨에 무관하게 항상 풀명 표시.
- 위험물제조소등: 줌 레벨이 7 이상(영종도 전체 보기 등)이면 라벨 숨김. 줌 인하면 다시 표시.
- 소방관서: 항상 표시. 줌 레벨 7 이상에서는 약식명("119안전센터" → "119" 등).

6. 마커 크기 규칙(위험물제조소등)
- 표준 템플릿 [지정수량 배수(합계)] 컬럼 값에 따라 상대 크기로 표시됩니다.
- 최소 업체를 1.0배(MIN 22px)로 두고, 큰 업체는 1+log10(value/min) 배수로 비례합니다.
- 최대 64px까지 도달하며, 그 이상은 같은 64px로 캡 처리됩니다.
- 컬럼 값이 비어 있는 업체는 최소 업체와 동일한 크기(22px)로 표시됩니다.

7. 관할구역 레이어
- 센터관할구역.geojson을 배경 경계 레이어로 상시 표시합니다.
- 외곽선만 옅게 표시되며, 선택된 소방관서가 있을 때 해당 관할이 강조됩니다.

8. 주의사항
- 저장되는 값은 카카오 JavaScript 키뿐입니다.
- 업로드한 엑셀 데이터는 브라우저 메모리에서만 처리하고 저장하지 않습니다.
- 모바일은 보조 조회용입니다.
- 카카오 JavaScript 키는 사용자 브라우저별로 저장되므로, 다른 PC나 다른 브라우저에서는 다시 입력해야 합니다.

9. 배포 트랙 (이 폴더는 GitHub용 소스)
- 본 폴더는 GitHub Pages 자동 배포의 소스 디렉터리입니다.
- 코드 수정 → GitHub Desktop으로 commit + push → 1~2분 후 GitHub Pages가 자동 갱신.
- 사용자에게는 GitHub Pages URL(예: https://USERNAME.github.io/yeongjong-dashboard/)만 즐겨찾기 안내.
- ZIP 로컬 배포가 필요한 경우 yeongjong-dashboard_ZIP배포용 폴더를 ZIP 압축해 전달.
- 두 트랙 모두 같은 카카오 JavaScript 키 사용 가능 (카카오 콘솔에 GitHub Pages 도메인 등록 필수).
