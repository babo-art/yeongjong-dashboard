# Map Icon Style Rules

이 문서는 카카오맵용 지도 아이콘을 추가하거나 수정할 때의 공통 규칙이다.

## 1. 목적

- 원본 아이콘의 의미는 유지한다.
- 지도 축소 상태에서도 읽히는 시인성을 확보한다.
- 테마가 바뀌어도 같은 규칙으로 아이콘 세트를 늘릴 수 있게 한다.

## 2. 기본 규격

- 출력 파일은 원본과 별도로 `-map.svg` 이름으로 저장한다.
- 캔버스는 `32 x 32`를 기본으로 한다.
- 실제 심볼은 `24 x 24` 기준 원본을 가운데 배치하고, 사방에 `4px` 여백을 둔다.
- 배경은 투명으로 유지한다.

## 3. 색상 규칙

- 기준지점 아이콘: `#2E7D32`
- 위험물제조소등 아이콘: `#D97706`
- 소방관서 아이콘: 119 표준색을 그대로 유지한다(`소방기관-map.svg` 본체 `#dc1f26`. 파일명은 옛 명칭 유지).
- `#8A0715`는 아이콘 본체 색이 아니라 `소방관서` 의미를 나타내는 보조 시각 요소(관할구역 외곽선, HUD 강조 등) 전용 색이다.
- 색상은 `currentColor`를 쓰지 않고 SVG 안에 고정한다.

## 4. 외곽선 규칙

- 모든 지도용 아이콘에는 흰색 외곽선을 넣는다.
- 외곽선 색상: `#FFFFFF`
- 외곽선 두께: `1.8`
- `stroke-linejoin="round"`를 기본으로 한다.
- 필요하면 `stroke-linecap="round"`도 추가한다.
- 보이는 심볼 path에만 외곽선을 적용한다.

## 5. 그림자 규칙

- 복잡한 지도 배경에서 묻히지 않도록 약한 그림자를 넣는다.
- 방법:
  - 동일 path를 한 번 더 복제한다.
  - 검정색 `#000000`
  - 불투명도 `0.18`
  - `translate(0 1)` 정도의 아래 그림자만 준다.
- 과한 blur filter는 피하고, 단순 shadow duplicate를 우선한다.

## 6. 구조 규칙

- 불필요한 편집기 클래스명은 제거해도 된다.
- `fill="none"`인 배경 초기화 path는 유지해도 무방하다.
- 실제 보이는 path는 그림자 path와 본체 path로 분리한다.
- 본체 path에는 `paint-order="stroke fill"`를 넣어 흰 테두리가 깔끔하게 보이게 한다.

## 7. 파일명 규칙

- 원본: `산(기준지점).svg`
- 지도용 편집본: `산(기준지점)-map.svg`

같은 방식으로:

- `소방기관-map.svg`
- `위험물제조소등-map.svg`

## 8. 카카오맵 적용 규칙

- 카카오맵 마커에 쓸 때는 이 `-map.svg`를 그대로 파일 경로로 쓰거나, 문자열로 읽어 data URI로 넣는다.
- 선택 상태 확대는 SVG를 다시 만들기보다 마커 크기만 키우는 방식이 우선이다.
- 선택 상태용 glow나 ring은 SVG 내부보다 지도 렌더링 로직에서 처리하는 것이 좋다.

## 9. 신규 테마 아이콘 추가 절차

1. 원본 SVG를 확보한다.
2. `32 x 32` 캔버스로 재배치한다.
3. 의미색을 정한다.
4. 흰색 외곽선을 추가한다.
5. 약한 그림자 duplicate path를 추가한다.
6. `-map.svg`로 저장한다.
7. 실제 지도 축소 상태에서 읽히는지 확인한다.

## 10. 이번 적용 파일

- `산(기준지점)-map.svg`
- `소방기관-map.svg`
- `위험물제조소등-map.svg`
